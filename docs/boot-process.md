# Linux Boot Flow

Mentat Linux currently uses the direct QEMU kernel boot path:

1. QEMU loads `out/bzImage` with `-kernel`.
2. QEMU passes `out/rootfs.cpio.gz` with `-initrd`.
3. The kernel decompresses itself and initializes early architecture code.
4. The kernel unpacks the initramfs into a temporary root filesystem in RAM.
5. The kernel executes `/init`.
6. `/init` becomes PID 1 and prepares early userspace.
7. PID 1 starts an interactive BusyBox shell.

The default command is generated by `scripts/run-qemu.sh`:

```sh
qemu-system-x86_64 \
  -m 512M \
  -kernel out/bzImage \
  -initrd out/rootfs.cpio.gz \
  -append "console=ttyS0 earlyprintk=serial,ttyS0,115200 panic=-1 init=/init" \
  -display none \
  -serial mon:stdio \
  -no-reboot \
  -netdev user,id=net0 \
  -device virtio-net-pci,netdev=net0
```

`console=ttyS0` sends kernel and userspace output to the serial console.

`earlyprintk=serial,ttyS0,115200` helps debug very early kernel output.

`panic=-1` prevents an immediate reboot after a kernel panic, which makes the
failure visible.

`init=/init` tells the kernel exactly which program should become PID 1.
